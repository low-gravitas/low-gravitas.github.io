# Memoized Fibonacci using functools
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n: int) -> int:
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

if __name__ == "__main__":
    for i in range(10):
        print(f"fib({i}) = {fib(i)}")

On fonts that age well

Typography choices that survive ten years of cultural churn, twenty years of browser-engine evolution, and one rebrand.

March 14, 2026 · by Mike Abney · 4 min read

Typography is the slowest-changing layer of a design system. Choose a face that won't embarrass you in five years — the wrong pick dates everything else around it. Look for typefaces that have already survived decades, not the ones trending on type sites this quarter.

What "aging well" means

Three things: stylistic restraint (no trend-chasing quirks), strong information architecture (a real weight range, proper italics, tabular figures), and institutional support — the foundry will still be here in ten years. Every webfont purchase is a bet that the servicing organization survives.

A good typeface is silent. It holds the content up without being the content.

The shortlist

I keep circling back to a handful of families that reliably look both current and timeless:

  • Fraunces — variable serif with warmth. Display and body out of one file.
  • IBM Plex — corporate-grade neutral sans with matching mono and serif siblings.
  • Söhne — if budget allows; the closest anyone's come to a modern Akzidenz.
  • Inter — ubiquitous but still safe, and it'll still be safe in 2035.

Using them well

Pairing a single display face with a quiet body face almost always beats a three-face system. Ship system fonts as the fallback, self-host the webfont, and use font-display: swap so your first paint isn't blocked.

css 
@font-face {
  font-family: "Fraunces";
  src: url("/vendor/fraunces.woff2") format("woff2-variations");
  font-display: swap;
  font-weight: 100 900;
}

body {
  font-family: "Fraunces", Georgia, serif;
  font-variation-settings: "opsz" 14, "SOFT" 100;
}
ⓘ
Heads up Variable fonts load as one file regardless of how many weights you use, but the browser still decodes the whole thing. Subset aggressively.

One caveat

Avoid display faces masquerading as body faces. A 48px headline is not a 16px paragraph. If the family doesn't ship an opsz axis or a separate "text" cut, pair a real text face with it and keep the display for headlines only.
